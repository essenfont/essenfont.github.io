# 15 тАФ SSG route generation (getStaticPaths)

The current site uses `scripts/gen-ssg-routes.mjs` to emit `public/ssg-routes.json` (consumed by vite-ssg's `includedRoutes`). Astro doesn't need this тАФ it uses `getStaticPaths` per-route.

But we still want a manifest of routes for sitemap + audit purposes. Keep `gen-ssg-routes.mjs` as the sitemap source, OR use `@astrojs/sitemap` which auto-discovers.

## Astro pattern

Each dynamic page implements `getStaticPaths`:

```astro
---
export async function getStaticPaths() {
  return blocks.map(b => ({ params: { blockSlug: b.slug }, props: { block: b } }));
}
---
```

Astro calls this at build time and emits one HTML file per returned path.

## Big-number consideration

- `/unicode/char/[hex]` alone generates ~160k pages
- Astro default `build.concurrency` may need tuning
- Consider `experimental.advancedRouting` or splitting char pages into multiple files (`[plane]/char/[hex].astro`) to keep build memory bounded

## Acceptance

- All routes pre-render to `dist/`
- Sitemap.xml regenerated by `@astrojs/sitemap`
- Page count matches `wc -l public/unicode/char/*` of the old build (parity check)
